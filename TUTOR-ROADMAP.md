# Tutor Roadmap (to-be)

แผนพัฒนาฝั่งติวเตอร์ — สิ่งที่ "ควรมีต่อ" มองจาก job-to-be-done ของติวเตอร์
อัปเดต: 2026-06-14 · สถานะ = ข้อเสนอ (ยังไม่เริ่ม) เว้นแต่ระบุไว้

effort: **S** เล็ก · **M** กลาง · **L** ใหญ่ · 🗄️ = ต้องมี migration

---

## ✅ มีแล้ว (baseline)

- **ข้อสอบ**: อัปโหลด HTML (kind ข้อสอบ/แบบฝึกหัด, subject, เวลา), card/table view, ค้นหา/เรียง/กรองตาม subject, แก้ชื่อ, แก้เวลา, เผยแพร่/ถอน, สลับให้ดูเฉลย, preview, ลบ
- **นักเรียน**: เพิ่มทีละคน/CSV/ไฟล์, ส่งรหัสครั้งเดียว, ค้นหา/เรียง, รีเซ็ตรหัส, แก้ชื่อ, ลบ, bulk
- **มอบหมาย**: drawer เลือกนักเรียน + ช่วงเวลา (เปิด/ปิด/กำหนดส่ง) + เวลา override + **ไม่จับเวลา** + undo
- **ผลสอบ**: tiles, anomaly banner, 3 มุมมอง (รายการส่ง/สรุปรายชุด/สรุปรายคน), วิเคราะห์รายข้อ, **แบบสอบถามฝั่งติวเตอร์**, **drill-down คะแนนรายคน**, เกณฑ์ผ่านต่อชุด, รีเซ็ตการทำ, export CSV, คะแนนเป็น %
- **โหมดทำข้อสอบ**: จับเวลา/ไม่จับเวลา/แบบฝึกหัด (ทำซ้ำ), แบบสอบถามหลังสอบ, ดูเฉลยครั้งเดียว (practice = ไม่จำกัด)

---

## Tier 1 — ปิด loop ที่ระบบชี้ปัญหาเองแต่ทำต่อไม่ได้ (ทำก่อน)

### 1. แก้/แทนเฉลย + คิดคะแนนใหม่ (fix key + re-grade) ⭐ สำคัญสุด · M 🗄️
- **ทำไม**: หน้าผลสอบมี banner "คะแนนต่ำผิดปกติ — อาจตรวจเฉลยเพี้ยน · ตรวจ mapping เฉลย" แต่กดได้แค่ดู **แก้ไม่ได้** → ถ้าเฉลยผิด คะแนนทุกคนผิดและติวเตอร์ติดตาย (เคยต้องแก้ผ่าน SQL มาแล้ว)
- **ขอบเขต**: แก้เฉลยรายข้อ / อัปโหลดทับชุดเดิม (คง exam id + ผลเดิม) แล้ว **re-grade attempts ที่ส่งแล้วอัตโนมัติ**
- **แตะ**: action แก้ `exam_answer_keys` + ฟังก์ชัน re-grade (อัปเดต `attempts.score`), UI ในหน้าผลสอบ/ข้อสอบ

### 2. กลุ่ม/ห้องเรียน (classes) · L 🗄️
- **ทำไม**: เมนูขึ้น "เร็วๆนี้" อยู่แล้ว · ติวเตอร์คิดเป็น "ห้อง" ไม่ใช่รายคน
- **ขอบเขต**: ตาราง group + group_members, มอบหมายทั้งห้องทีเดียว, กรองผลสอบ/นักเรียนตามห้อง
- **คูณค่า**: ยกระดับทั้งหน้ามอบหมาย + ผลสอบ + นักเรียน

### 3. คลังข้อสอบ / archive · S 🗄️(พร้อมแล้ว)
- **ทำไม**: เมนู "เร็วๆนี้" + **DB รองรับแล้ว** (`exams.status='archived'` ใน migration 0005) เหลือแค่ UI
- **ขอบเขต**: เก็บชุดเก่าโดยไม่ลบ (ซ่อนจากรายการหลัก, เรียกคืนได้) — effort ต่ำสุดใน Tier 1

### 4. คัดลอกชุด (duplicate) · S
- **ทำไม**: เมนู ⋯ "เร็วๆนี้" อยู่แล้ว · ทำชุด variant / นำกลับมาใช้
- **ขอบเขต**: copy exam + answer key เป็นชุดใหม่ (draft)

## Tier 2 — ยกระดับการวิเคราะห์เชิงสอน

### 5. tag หัวข้อรายข้อ → mastery รายหัวข้อ · L 🗄️
- **ทำไม**: item analysis เป็น "ข้อที่ N" แต่ติวเตอร์คิดเป็น "อ่อนเรื่องกลศาสตร์/ไฟฟ้า" (survey ถาม "บทยากสุด" แบบ free-text อยู่แล้ว — อันนี้คือเวอร์ชันมีโครงสร้าง)
- **ขอบเขต**: tag ข้อเข้าหัวข้อ → สรุปจุดอ่อนรายหัวข้อ ทั้งห้อง + รายคน

### 6. กราฟพัฒนาการรายคนตามเวลา · M
- **ทำไม**: `StudentSummary.trend` (ดีขึ้น/แย่ลง) เป็นเชื้ออยู่แล้ว
- **ขอบเขต**: กราฟคะแนน % ข้ามชุดเรียงเวลา ใน drawer รายคน (ที่มีอยู่)

### 7. รายงานรายคนแบบพิมพ์/ส่งผู้ปกครอง (PDF/printable) · M
- **ทำไม**: มี CSV ทั้ง gradebook แล้ว ขาดเวอร์ชันรายคนที่ส่งต่อได้

---

## Tier 3 — สื่อสาร / ดำเนินการ

### 8. เตือนนักเรียนที่ยังไม่ทำ · S–M
- **ทำไม**: dashboard ตรวจเจอ "ยังไม่เปิดลิงก์" แล้ว แต่กดทำอะไรไม่ได้
- **ขอบเขต**: ปุ่มคัดลอกข้อความเตือน / mark (ไม่มี push/email infra → copy-to-clipboard ก่อน)

### 9. ฟีดแบ็กถึงนักเรียนต่อครั้ง · M 🗄️
- **ทำไม**: survey มี "ถึงติวเตอร์" (เด็ก→ครู) แต่ไม่มีทางตอบกลับ
- **ขอบเขต**: ติวเตอร์คอมเมนต์ต่อ attempt → นักเรียนเห็นในหน้าผล

### 10. ตั้งค่า/โปรไฟล์ติวเตอร์ · S
- **ทำไม**: เมนูบัญชี "เร็วๆนี้" (เปลี่ยนรหัส/ชื่อติวเตอร์ ฯลฯ)

## ลำดับที่แนะนำ
1. **fix key + re-grade** — ความถูกต้อง/ความเชื่อใจ สำคัญกว่าทุกอย่าง
2. **กลุ่ม/ห้องเรียน** — คูณค่าทั้งระบบ
3. **archive** — effort ต่ำ DB พร้อม (ทำแทรกได้เร็ว)

> หมายเหตุ: migration ต้องรันมือใน Supabase SQL Editor (workflow ปัจจุบัน) — ดูไฟล์ `supabase/migrations/`
