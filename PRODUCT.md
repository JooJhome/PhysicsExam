# Product

## Register

product

## Users

**ผู้ใช้หลัก: นักเรียนระดับ ม.ปลาย** ที่เข้ามาทำข้อสอบฟิสิกส์ปรนัย 30 ข้อ
(แต่ละชุดทำได้ครั้งเดียว มีการจับเวลา) — มักเป็นช่วงเวลาที่กดดันและมีสมาธิจดจ่อ
หลายคนเข้าจากมือถือ บริบทคือ "กำลังจะสอบ/กำลังสอบ" จึงต้องการความชัดเจน ไม่มีสิ่งรบกวน
และความมั่นใจว่าระบบทำงานถูกต้อง (คำตอบถูกบันทึก เวลาเที่ยงตรง คะแนนยุติธรรม)

**ผู้ใช้รอง: ติวเตอร์/แอดมิน** ที่จัดการชุดข้อสอบ สร้างบัญชีนักเรียน มอบหมายชุด
และติดตามผลแบบเรียลไทม์ บริบทคือการทำงานบน desktop เน้นประสิทธิภาพ ดูข้อมูลหลายคนพร้อมกัน
และจัดการเคสเฉพาะ (เช่น รีเซ็ตให้ทำใหม่เมื่อเน็ตหลุด)

**Job to be done:** นักเรียน — สอบให้เสร็จอย่างราบรื่นและรู้ผล/เฉลยทันที;
ติวเตอร์ — ออก/มอบหมายข้อสอบและอ่านผลได้เร็วโดยเฉลยไม่รั่ว

## Product Purpose

ระบบสอบฟิสิกส์ออนไลน์สำหรับห้องเรียนขนาดเล็ก (~30 คน) ที่ทำให้การสอบปรนัยเป็นเรื่องน่าเชื่อถือ
และยุติธรรม: ตรวจคะแนนฝั่งเซิร์ฟเวอร์เพื่อกันเฉลยรั่ว บังคับทำได้ครั้งเดียวที่ระดับฐานข้อมูล
มีลายน้ำกันไฟล์หลุด และให้นักเรียนเห็นเฉลยละเอียดหลังส่งเพื่อการเรียนรู้

**ความสำเร็จ** = นักเรียนทำข้อสอบจบโดยไม่สะดุดและไว้ใจผลคะแนน · ติวเตอร์จัดการทั้งวงจร
(ออก → มอบหมาย → อ่านผล) ได้โดยไม่ต้องใช้เครื่องมือเสริม · เฉลยไม่เคยรั่วก่อนส่ง

## Brand Personality

**สามคำหลัก: น่าเชื่อถือ · สงบ · คมชัด**

- **น่าเชื่อถือ จริงจัง** — นี่คือการสอบจริง ระบบต้องดูเชื่อมือ ไม่เล่นๆ ผลคะแนนน่าไว้ใจ
- **สงบ ลดความกดดัน** — เป้าหมายผู้ใช้คือเด็กที่กำลังเครียดกับการสอบ UI ต้องไม่เพิ่มความวิตก
  หน้าจอโปร่ง โฟกัสทีละสิ่ง ไม่มีสิ่งรบกวนระหว่างทำข้อสอบ
- **ทันสมัย คมชัด** — typography และ spacing ที่ประณีต ไม่ดูเหมือนระบบราชการเก่า
- **เป็นกันเอง อบอุ่นพอประมาณ** — ภาษาไทยที่สุภาพแต่ไม่ห่างเหิน ให้กำลังใจในจุดที่ควร
  (เช่น หน้าผลคะแนน เฉลย) โดยไม่กลายเป็นเกมหรือการ์ตูน

**โทนภาษา:** ภาษาไทยกระชับ ชัดเจน ใช้คำสั่งตรงไปตรงมา ("เริ่มทำ" "ส่งคำตอบ")
ในจุดที่มีความเสี่ยง (ส่งแล้วแก้ไม่ได้ / ทำได้ครั้งเดียว) พูดให้ชัดล่วงหน้าอย่างใจเย็น ไม่ขู่

**Visual reference:** สไตล์แนว EduLearn — เทียลเข้ม + อำพันอุ่น + พื้นครีม + ตัวอักษรหนาเป็นมิตร
+ ลูกเล่นวาดมือ (เส้นใต้/ดาว/ดูเดิล) อบอุ่นเป็นกันเองในหน้าทั่วไป แต่สงบเรียบในหน้าทำข้อสอบ
รายละเอียดภาพอยู่ใน [DESIGN-SYSTEM.md](DESIGN-SYSTEM.md)

## Anti-references

- **ระบบราชการ/e-learning เก่า** — ตารางแน่นๆ สีน้ำเงินเข้มทึบ ฟอนต์เล็ก กรอบหนา เมนูซ้อนลึก
- **การ์ดเหมือนกันเรียงเป็นกริด** — กล่องสี่เหลี่ยมขนาดเท่ากันไอคอน+หัวข้อ+ข้อความซ้ำๆ ทั้งหน้า
- **ฉูดฉาด เกมเกินไป** — สีสด แอนิเมชันเด้งดึ๋ง gamification badge/แต้ม ที่ลดความน่าเชื่อถือของการสอบ
- **จืดชืด ไม่มีเอกลักษณ์** — ดีไซน์ default ที่ดูเหมือน template Tailwind ทั่วไป ไม่มีบุคลิกของแบรนด์

## Design Principles

1. **โฟกัสคือความเมตตา** — ระหว่างทำข้อสอบ ตัดทุกอย่างที่ไม่ใช่คำถามและคำตอบออก
   ลดภาระทางความคิด (cognitive load) ให้เหลือสิ่งที่ผู้สอบต้องตัดสินใจเท่านั้น
2. **ความชัดเจนสร้างความไว้ใจ** — สถานะของทุกอย่างต้องอ่านออกทันที (ทำต่อ/ส่งแล้ว/หมดเวลา,
   เหลือกี่ข้อ, เวลาที่เหลือ) ไม่มีสภาวะที่ผู้ใช้เดาว่าเกิดอะไรขึ้น
3. **สงบแต่จริงจัง** — ดีไซน์เงียบ มีพื้นที่หายใจ แต่ยังหนักแน่นพอให้รู้สึกว่าเป็นการสอบจริง
   ไม่ใช่แบบฝึกหัดเล่นๆ
4. **บอกล่วงหน้าก่อนจุดที่ย้อนไม่ได้** — การกระทำที่ทำครั้งเดียว (เริ่มสอบ ส่งคำตอบ ดูเฉลย)
   ต้องสื่อสารผลลัพธ์ให้ชัดและใจเย็นก่อนเสมอ
5. **มือถือคือพลเมืองชั้นหนึ่ง** — ทุกหน้านักเรียนต้องใช้งานได้ดีบนจอเล็ก ปุ่มกดง่าย
   อ่านสบายตา ก่อนค่อยขยายไป desktop

## Accessibility & Inclusion

- เป้าหมาย **WCAG AA**: คอนทราสต์ตัวอักษรเนื้อหา ≥ 4.5:1, ตัวใหญ่ ≥ 3:1 — เลี่ยงเทาอ่อนบนพื้นสีอ่อน
- **อ่านง่ายเป็นหลัก** — เน้นขนาดตัวอักษรพอเหมาะ ระยะบรรทัดสบายตา โดยเฉพาะตัวข้อสอบและเฉลย
- **ใช้บนมือถือได้ดี** — touch target ≥ 44px, ไม่ต้องซูม, จัดวางให้นิ้วถึงปุ่มสำคัญง่าย
- **รองรับ `prefers-reduced-motion`** — ทุกแอนิเมชันต้องมีทางเลือกแบบ crossfade/instant
- ยังไม่มีข้อกำหนดเฉพาะด้าน screen reader หรือ color blindness แต่ออกแบบให้ไม่พึ่งสีเพียงอย่างเดียว
  ในการสื่อความหมาย (ถูก/ผิด ใช้ไอคอน+ข้อความควบคู่สี)
